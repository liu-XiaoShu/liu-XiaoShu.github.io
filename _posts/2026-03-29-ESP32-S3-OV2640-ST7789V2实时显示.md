# ESP32-S3 + OV2640 + ST7789V2：摄像头实时预览

**项目状态**： ✅ 硬件已验证 | ✅ 固件可运行 | 📖 代码与 README 已开源

### 📋 项目简介

![实现的效果实物图](/assets/img/esp32-s3-ov2640_st7789v2/实现的效果实物图.jpg)
![实现的效果实物图-2](/assets/img/esp32-s3-ov2640_st7789v2/实现的效果实物图-2.jpg)

本项目基于 **PlatformIO + Arduino**，在 **ESP32-S3** 上将 **OV2640** 摄像头采集的图像实时显示到 **ST7789V2**（**240×280**，4 线 SPI）液晶屏上。画面默认以 **QQVGA** 采集、**RGB565** 格式，经预计算的 **xMap / yMap** 做最近邻缩放到 **240×240** 显示区（竖屏居中），**右上角叠加 FPS**，便于观察流畅度。

设计取向：**无 WiFi、无网页**，只做本地预览，优先帧率与代码路径清晰。完整工程见 GitHub：**[esp32s3-ov2640-to-st7789v2](https://github.com/liu-XiaoShu/esp32s3-ov2640-to-st7789v2)**。

---

### 🎯 核心特性

* **OV2640 实时采集**：`RGB565`，默认 `FRAMESIZE_QQVGA`，可改分辨率权衡算力与 FPS。
* **ST7789V2 4-Wire SPI**：分辨率 **240×280**，竖屏；SPI 默认 **40 MHz**（花屏可适当降低）。
* **FPS 叠加**：在画面区域右上角绘制 `FPS: xx.x`，半透明色键与 Adafruit `GFXcanvas16` 合成。
* **缩放优化**：预计算 `xMap[]` / `yMap[]` 最近邻映射，避免每帧重复除法，降低单帧开销。
* **抗运动模糊倾向**：通过限制增益上限、`ae_level` 偏短曝光等方式减轻暗光下拖影（液晶响应时间仍有物理限制）。
* **可选自检**：`DISPLAY_SELF_TEST` 开机 RGB 全屏自检；`CAMERA_COLORBAR_TEST` 传感器彩条（调试用）。

### 🔌 硬件与接线

摄像头引脚采用 **Freenove ESP32-S3 WROOM** 与 `esp32-camera` 中 **ESP32S3_EYE** 一类定义（见仓库 `main.cpp`）。显示屏与 **ESP32-S3** 连接如下（与 README 一致）：

| 屏幕引脚 | ESP32-S3 引脚 |

| GND | GND |
| VCC | 3V3 |
| SCL | GPIO40 |
| SDA | GPIO39 |
| RES | GPIO38 |
| DC  | GPIO42 |
| CS  | GPIO41 |
| BLK | GPIO47 |

### ⚙️ 关键参数（`src/main.cpp`）

| 参数/宏 | 说明 |
| :--- | :--- |
| `CAM_FRAME_SIZE` | 默认 `FRAMESIZE_QQVGA`；改为 `QVGA` 等会提高负载、可能影响 FPS |
| `lcdSPI.setFrequency` | 默认 40 MHz；花屏可略降 |
| `CAM_RGB565_SWAP_BYTES` / `CAM_RGB565_IS_BGR` | 颜色异常时逐项调整 |
| `CAM_COLOR_TUNE` | 每像素微调颜色，默认关闭以利于帧率 |
| `DISPLAY_SELF_TEST` | 开机全屏 RGB 自检，默认关 |
| `CAMERA_COLORBAR_TEST` | 传感器彩条测试 |
| `CAM_AE_LEVEL_BIAS` / `CAM_GAIN_CEILING` | 曝光与增益，影响亮度与运动模糊 |

### ❓ 常见问题

**串口 `/dev/ttyACM0 Permission denied`**

```bash
sudo usermod -aG dialout,uucp $USER
```

重新登录后再试。

**彩条正常、实景偏色**

在 `CAM_RGB565_SWAP_BYTES`、`CAM_RGB565_IS_BGR` 上组合尝试，直至 RGB565 字节序与 BGR/RGB 与传感器输出一致。
