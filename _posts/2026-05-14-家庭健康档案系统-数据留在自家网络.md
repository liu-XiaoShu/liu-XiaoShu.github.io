# 家庭健康档案系统（Family Health Archive）

**项目状态**： ✅ 前后端可运行 | ✅ 需求与 README 已整理 | 📖 源码与协议已开源

### 📋 项目简介

![首页](/assets/img/family-health-archive/home_page.png)

体检单、化验单和各家医院 App 里的结果常常**零散难找**；家里老人多份报告想**按人、按时间对照**，光靠纸质和相册也很费劲。  
**家庭健康档案系统**把结构化数据落在**本机 SQLite**，用浏览器在**家庭或可信局域网**里访问——适合「**数据不出自家网络**」的自我管理场景，而不是把敏感健康数据交给第三方云端。

完整工程与文档见 GitHub：**[xiaoshuFamilyHealthArchive](https://github.com/liu-XiaoShu/xiaoshuFamilyHealthArchive)**。

---

### 🎯 核心特性

* **家庭成员档案**：基础信息、头像；部分字段可由报告回填。
* **报告批次**：按一次体检/复查归组，明细挂在批次下，避免「一堆散项」。
* **指标体系**：树形指标（分组 + 叶子项），器官库与自选器官，便于和检查报告口径对齐。
* **录入与多视图**：按人、按批次、按器官或类型、按时间梳理录入与查阅。
* **趋势图**：基于 **Chart.js** 的时间序列与标注，方便跨年对比（个人备忘视角）。
* **导入导出**：CSV 模板导出与批量导入（UTF-8，建议带 BOM）。

### 🚀 本地体验（极简）

**克隆与依赖**（首次需 Python 3.8+、Node LTS）：

```bash
git clone git@github.com:liu-XiaoShu/xiaoshuFamilyHealthArchive.git
cd xiaoshuFamilyHealthArchive

cd backend && python3 -m venv .venv && ./.venv/bin/pip install -r requirements.txt && cd ..
cd frontend && npm install && cd ..
```

**一键开发启动**（项目根目录）：

```bash
chmod +x scripts/start-dev.sh
export PYTHON_BIN="$(pwd)/backend/.venv/bin/python"
./scripts/start-dev.sh
```

浏览器访问前端：**http://127.0.0.1:5173**（开发模式下 API 由 Vite 代理到后端）。  
数据库为空时，会按环境变量创建首个管理员；默认账号口令见仓库 README，**生产环境请务必修改**并在 `backend/.env` 中配置 `FH_SESSION_SECRET` 等。

---

### 🏠 局域网部署（思路）

推荐 **构建前端 + 由后端同端口托管静态资源**，保证 Cookie 与上传（如 CSV）同源、配置简单：

```bash
cd frontend && npm run build && cd ../backend
./.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
```

局域网内用 `http://<服务器IP>:8000` 访问；数据库文件默认在 `backend/data/family_health.db`，**备份与迁移前请先停服务或做好文件拷贝**。
